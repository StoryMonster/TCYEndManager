环境依赖

  • python 3.x with tkinter installed
  • windows x86_64

    • 启动方式（两种启动方式没有区别）

  • 运行 TCYServersManager.bat 启动
  • 运行 TCYServersManager.ps1 启动

    • 功能

  • 控制服务器启动
  • 控制移动端模拟器启动
  • 快捷编译服务端程序

    • 配置文件 config.xml

    servers

    使用者可以根据自己的需求按照类似格式添加server，server各属性字段讲解如下：

    compiler[optional]: 编译server所要求的编译环境，当前只支持Visual Stuido的vcvars*.bat

  • compile_env_config_file: Visual Studio的vcvars*.bat文件
  • compile_enabled_servers: 支持编译的server，server名用分号分隔开，不要插入额外空格

    • quick_launch_servers: 一键启动服务器

  • launch_queue: 服务器的启动队列，名称用分号隔开，不要插入额外空格

    • instances: 具体的服务器实例

  • exe_file: 可执行文件路径，要求所有启动相关配置文件都要和它在同一目录
  • is_window_expected: 是否为此服务器开启窗口

    • clients

    simulator[mandantory]:

  • path: 移动端模拟器路径
  • script: 移动端的逻辑代码

    • instances: 具体的clients实例

  • is_console_expected: 是否希望在模拟器运行时为该client实例显示log控制台
  • account: 同城游账号
  • password: 同城游密码

    • 注意事项

  • 按照自己的实际环境修改config.xml
  • config.xml中所有路径使用绝对路径
  • 如果本地powershell运行脚本权限未开启，可以选择使用TCYServersManager.bat启动

    • 版本特色

  • 精简配置文件，使用户使用更简便，但是相对增加了对用户本地环境的要求
  • 服务器运行前会先检查系统中是否有相同进程存在，强制每个服务器单例运行
  • 支持路径中使用“\”，拷贝地址时不用做额外修改
  • 只有开启窗口的进程的运行log会被显示在窗口中，其他进程（除了编译）的log将不在Common Window中显示
  • 开启窗口的进程支持右键跳转到运行目录，方便提取运行log
  • 优化了使用中的一些影响体验的功能模块
